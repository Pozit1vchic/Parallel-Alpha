# Правила для AI-ассистента (Parallel-Alpha)

## 🚨 КРИТИЧЕСКИ ВАЖНО
Ты работаешь над проектом **Parallel-Alpha** — инструментом для видео-монтажёров на базе Computer Vision.

### 1. Импорты и зависимости
- **ЗАПРЕЩЕНО** придумывать библиотеки, которых нет в `requirements.txt` или в импортах существующих файлов.
- Используй **ТОЛЬКО**: `ultralytics`, `torch`, `numpy`, `faiss-cpu`, `opencv-python`, `tkinter`, `PIL`, `json`, `csv`.
- **НЕ ИСПОЛЬЗУЙ** `torch.compile` (багован на Windows с FAISS).
- **НЕ ИСПОЛЬЗУЙ** `asyncio` в связке с Tkinter (вешает GUI).

### 2. Стиль кода
- Названия классов: **CamelCase** (например, `VideoProcessor`).
- Названия функций и переменных: **snake_case** (например, `extract_keypoints`).
- Комментарии: **на русском или английском**, кратко, только суть алгоритма.
- Типизация: **обязательна** (`def process(self, frame: np.ndarray) -> list[float]:`).

### 3. Оптимизация (RTX 5070 / 12GB VRAM)
- При работе с PyTorch всегда используй `torch.inference_mode()` вместо `torch.no_grad()`.
- Для FAISS индекс всегда создавай на CPU (`faiss.IndexFlatL2`), НЕ пытайся использовать GPU-индексы (они под Windows нестабильны).
- При обработке видео используй `torch.cuda.empty_cache()` каждые 500 кадров.

### 4. GUI (Tkinter)
- **НЕ МЕНЯЙ** геометрию `pack()` на `grid()` без явной команды.
- Все длительные операции (загрузка видео, анализ) выполняй в **отдельном потоке** (`threading`), иначе интерфейс зависнет.
- Для отображения прогресса используй `ttk.Progressbar`.

### 5. Ответы на вопросы
- Если просят написать код — **сразу давай код**, без длинных предисловий.
- Если просят объяснить ошибку — сначала **покажи место в коде**, где она возникает, потом решение.
- Предлагай **векторизованные операции** (NumPy/PyTorch) вместо циклов Python.

### 6. Контекст проекта
- `core/video_processor.py` — основной движок анализа (YOLO + FAISS + Soft-DTW).
- `ui/main_window.py` — GUI на Tkinter.
- `utils/file_handler.py` — работа с JSON/CSV/EDL экспортом.
- `config.json` — настройки путей и порогов схожести.
